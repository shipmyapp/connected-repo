# Shared Zod Schemas Package (AGENTS.md)

## 1. Blueprint
**Intent**: Centralized, type-safe validation layer for backend/frontend consistency.
**Core Stack**: Zod 3, TypeScript 5.

**Key Architectures**:
- **Entity Pattern**: Strict 7-component structure for every entity (Mandatory, Optional, AutoGenerated, CreateInput, UpdateInput, SelectAll, Query).
- **Indian Compliance**: Specialized validators for GSTIN, PAN, Udyam, etc.
- **DB-Aligned**: Numeric and string validators matched to PostgreSQL/Orchid ORM constraints.
- **Type Inference**: Automatic TS type extraction from Zod schemas for E2E safety.

## Technical Guidelines

### Entity Pattern (MANDATORY)
```typescript
export const entityMandatoryZod = z.object({ name: zString.min(1) });
export const entityOptionalZod = z.object({ description: zString.nullable() });
const entityAutoGeneratedZod = z.object({ entityId: z.uuid() }); // DB-managed

export const entityCreateInputZod = entityMandatoryZod.extend(entityOptionalZod.partial().shape);
export const entityUpdateInputZod = entityMandatoryZod.extend(entityOptionalZod.shape).extend(zTimestamps).partial();
export const entitySelectAllZod = entityMandatoryZod.extend(entityOptionalZod.shape).extend(zTimestamps).extend(entityAutoGeneratedZod.shape);
```

### Shared Validators (`zod_utils.ts`)
- **Strings**: `zString`, `zVarchar(max)`, `zText()`.
- **Numbers**: `zInteger()`, `zBigint()`, `zPrice` (Decimal 10,2), `zQuantity` (Decimal 11,3).
- **Compliance**: `zGSTIN`, `zPAN`, `zPhoneNumber`.
- **System**: `zTimestamps`, `NodeEnv`.

### Usage
```typescript
import { entityCreateInputZod } from '@connected-repo/zod-schemas/entity.zod';
import { zString, zTimestamps } from '@connected-repo/zod-schemas/zod_utils';
```

### Best Practices
- **YES** use `.safeParse()` for runtime checks.
- **YES** omit sensitive fields (e.g., `apiSecretHash`) in `SelectAllZod`.
- **NO** use `any` or manual type assertions.
- **NO** package root imports; use direct module paths.
